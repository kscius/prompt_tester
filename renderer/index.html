Temperatura del modelo

Qué hace. La temperatura controla cuánta aleatoriedad usa el modelo al elegir cada palabra (token). Con valores bajos tiende a la respuesta más probable y predecible; con valores altos explora opciones menos probables y obtienes más variedad (a veces más creatividad, a veces más errores o divagaciones).

Cómo usar el control (0 → 2).

  • 0 – 0,3 — Casi determinístico. Útil para clasificar, extraer datos, JSON o formatos rígidos, y cuando necesitas resultados repetibles al reenviar el mismo prompt.
  • 0,5 – 0,8 — Conservador y estable. Buen punto de partida para respuestas técnicas o asistentes que deben ceñirse al contexto.
  • 1,0 — Valor equilibrado por defecto en muchos flujos; buen balance entre coherencia y naturalidad.
  • 1,2 – 2,0 — Más creatividad y variación entre envíos. Sirve para lluvia de ideas, storytelling, variaciones de tono o copia; sube el riesgo de inventar detalles o salirse del guion.

Ejemplo (mismo prompt, distinta temperatura). Si pides «un eslogan corto para una cafetería», con temperatura baja sueles obtener frases genéricas y parecidas entre una ejecución y otra; con temperatura alta cada respuesta puede cambiar mucho (juegos de palabras distintos, tono más informal o más poético). No garantiza “mejor” texto: solo más o menos variedad.

Nota. La temperatura no sustituye instrucciones claras. Si ves alucinaciones o incoherencia, prueba bajar la temperatura y/o acotar el prompt con datos y formato esperados.
