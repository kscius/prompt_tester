# Contribuir

1. Abre un issue para bugs o ideas grandes antes de invertir mucho tiempo, salvo que sea un arreglo trivial.
2. Haz un fork, crea una rama desde `main` y envía un pull request con una descripción clara de qué cambia y por qué.
3. Ejecuta la app con `npm start` y, si tocas el empaquetado, prueba al menos el comando de build de tu sistema operativo.
4. No incluyas credenciales, claves ni datos personales en commits, capturas o archivos del repo.

## Estilo de código

- Respeta el estilo ya presente en `main.js`, `preload.js` y `renderer/`.
- Cambios pequeños y enfocados suelen revisarse antes que refactors amplios sin consenso.
